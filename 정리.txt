## Git 기초 명령어
  - clone : 원격 저장소 복사
  - add : 스테이지 영역에 작업 파일 추가하기
  - push : 원격 저장소에 커밋으로부터
  
## 파일 내용 되돌리기
  - 해당 파일 선택 후 <코드 뭉치 버리기> 
  
## 브랜치 변경
  - 브랜치 : 기존 내용을 유지한 채 새로운 내용을 추가할 때 브랜치를 생성
  - 체크아웃 : 특정 브랜치로 돌아가고 싶을 때 사용
  - 소스트리에서는 브랜치 이름을 더블 클릭하면 체크아웃 가능
  
## 병합하기1
  - 병학 대상이 헤드로부터 시작된 경우 
  - fast-forward 병합하기1
  - git 2.2 버전부터 merge 방식 지정을 해줘야함
  
## 병합하기2
  - 헤드 브랜치에서 추가적인 커밋이 생기면
  - 수동으로 편집기를 열어서 병합
  - 다양한 diff tool 이용하기(acroedit, notepad++, 다양한 IDE 등)
  
## 충돌 해결
  - 같은 파일을 두 커밋에서 동시 수정했을 경우 merge가 제대로 안될 수 있음
  - diff tool 사용하거나 소스트리에서 충돌 처리 할 것

## 커밋 되돌리기

## Reset
  - 제일 심플방법 
  - 되돌린 이후 시점의 커밋이 날아감
  - 강제 푸시 (git push --force)
  
## Branch 만들어서 되돌리기
  - reset 의 경우 커밋이 날아가지만 branh를 만들면 커밋을 살릴 수 있음
  - 비교적 사용이 쉬운편
  - 트리가 복잡해짐
  
## Revert 사용하기
  - 커밋은 사라지지 않음
  - 정석적인 방법
  - 충돌이 자주 발생할 수 있음

## stash
  - 다른브랜치로 체크아웃하기 전 임시 저장소
  - 자주 사용해 보자
  
## Rebase
  - merge랑 유사하게 두 브랜치를 병합
  - 현재 브랜치가 대상 브랜치 위로 올라감
